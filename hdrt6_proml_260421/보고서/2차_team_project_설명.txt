2차 팀 프로젝트 260421-260428

중간 프로젝트 : 머신러닝 기반 데이터 분석 프로젝트 수행 안내
프로젝트는 그동안 학습한 머신러닝 지식을 바탕으로, 실제 현업에서 수행하는 데이터 기반 문제 해결 과정을 경험하는 것을 목표로 합니다. 
각 팀은 주어진 기간 동안 하나의 주제를 선정하여 데이터 수집부터 전처리, 모델 설계, 성능 평가, 결과 해석 및 개선 방안 도출까지의 전 과정을 독립적으로 수행해야 합니다.

프로젝트는 단순한 모델 구현을 넘어서, 문제 정의의 명확성, 데이터의 적절성, 전처리 과정의 타당성, 모델 선택의 논리성, 평가 지표의 적절성 등을 종합적으로 고려하여 진행되어야 합니다.  특히 가설검정, Regression 또는 Classification 중, 해당 문제에 적합한 모델을 비교·분석하고 최적의 결과를 도출하는 과정이 포함되어야 합니다.

데이터는 공공데이터, 오픈 데이터셋, 크롤링 등을 통해 직접 확보하거나, 기존 데이터를 가공하여 사용할 수 있으며, 결측치 처리, 이상치 제거, 인코딩, 스케일링 등 전처리 과정에 대한 명확한 근거와 설명이 포함되어야 합니다. 
또한 단일 모델에 의존하기보다는, 여러 모델을 비교하거나 앙상블 기법을 적용하는 등 성능 개선을 위한 시도가 요구됩니다.

모델 학습 후에는 Accuracy, Precision, Recall, RMSE, ROC-AUC 등 문제 유형에 적합한 평가 지표를 활용하여 성능을 검증하고, 결과에 대한 해석을 제시해야 합니다. 
더 나아가, 모델의 한계점 분석 및 성능 개선을 위한 추가적인 전략(하이퍼파라미터 튜닝, Feature Engineering 등)을 포함하여 실무적인 관점에서 접근해야 합니다.

최종 결과물에는 코드, 실행 결과, 시각화 자료, 분석 내용이 포함되어야 하며, 팀원 간 역할 분담과 협업 과정 또한 드러나야 합니다. 

협업 방식 (추천)
 1. 데이터 공유 (공통 폴더)
 2. 전처리 → 모델 → 평가 순서로 협업
 3. 중간 점검 (1일 1회)
 4. 마지막 날 통합 및 발표 준비

문제 정의 → 데이터 → 모델 → 평가 → 개선 → 해석 과 같은 흐름이 보이면 좋은 프로젝트입니다~~~

실무형 팀 프로젝트 주제 -----
1. 고객 이탈(Churn) 예측 시스템 : 고객이 서비스를 떠날 가능성 예측
2. 신용카드 사기 거래 탐지 : 정상 거래 vs 사기 거래 분류
3. 중고차 가격 예측 : 차량 정보로 가격 예측
4. 제품 수요 예측 (Inventory Forecasting) : 목표: 특정 제품의 판매량 예측
5. 영화 추천 시스템 (간단 추천 모델) : 사용자 취향 기반 영화 추천
6. 온라인 쇼핑 구매 여부 예측 : 방문자가 구매할지 예측
7. 병원 환자 질병 예측 : 증상 기반 질병 분류
8. 직원 이직 예측 (HR Analytics) : 직원 퇴사 가능성 예측
9. 배달 주문 수요 예측 : 시간/날씨 기반 주문량 예측
10. 보험 가입 여부 예측 (Insurance Conversion) : 고객이 보험 상품에 가입할지 예측
     데이터: 고객 나이, 직업, 소득, 기존 보험 여부

5일 프로젝트 일정표 예 ------------------------------------------
1) 문제 정의 & 데이터 확보
   핵심: “무엇을 해결할 것인가?” 프로젝트 주제 확정, 문제 정의 (Classification / Regression 명확히 구분)
   데이터 수집 (공공데이터, Kaggle, 크롤링 등), 데이터 구조 파악 (컬럼 의미 정리)
   결과물 : 프로젝트 기획서 초안, 데이터 설명 문서
2) 데이터 전처리 & EDA
   핵심: “데이터를 이해하고 정리”, 결측치 처리 (삭제 / 대체), 이상치 탐지 및 처리, 범주형 인코딩 / 스케일링, EDA (시각화, 상관관계 분석)
   결과물 : 전처리 코드, EDA 시각화 (그래프 최소 3~5개)
3) 모델링 & 초기 성능 확인
   핵심: “기본 모델 만들기” : Baseline 모델 생성, 다양한 모델 비교 (최소 2~3개), train/test split, 기본 성능 평가
   결과물 : 모델 비교 결과 표, 초기 성능 지표 (Accuracy, RMSE 등)
4) 성능 개선 & 튜닝
   핵심: “성능을 끌어올리기” : GridSearchCV / RandomSearch 적용, Feature Engineering, 앙상블 적용, 교차 검증 수행
   결과물 : 최적 모델, 성능 개선 전/후 비교
5) 결과 정리 & 발표 준비
   핵심: “설명 가능해야 한다” : 결과 해석 (왜 이렇게 나왔는가?), 중요도 분석, 모델 한계점 도출, 개선 방향 제시, 발표 자료 제작
   결과물 : 발표 자료, 최종 코드 정리, 프로젝트 보고서

좋은 프로젝트는 “왜?”라는 문제 제시와 그에 대한 답이 있어야 합니다.

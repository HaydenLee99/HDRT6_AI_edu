팀명 : 4팀
팀장 : 이희동
팀원 : 김동하, 박상우, 이다현, 한재현

제목 : ArduPilot SITL 환경에서 3DR Iris+ Quadcopter 배터리 소모량 예측.
	- 비행 환경 데이터 기반, 시뮬레이션 드론 배터리 소모량 예측을 통해 비행 가능 여부를 판단하는 머신러닝 기반 의사결정 시스템

1. 개요
본 프로젝트는 ArduPilot SITL 및 3DR Iris+ 드론 시뮬레이션 환경을 활용하여 비행 데이터를 수집하고,
센서 및 비행 상태 데이터를 기반으로 드론의 배터리 소모량을 예측하는 머신러닝 시스템 구축을 목표로 한다.

해당 시뮬레이션에는 배터리 소모에 대한 정보가 없으므로 저전력 상황으로 인한 RTH 발생 등 이슈를 파악하기 어렵다.
이를 시뮬레이션 환경에서도 알고자 비행 log를 기반으로 소모량을 수치화하고, 
이를 웹으로 드론 이용자가 원하는 고도와 경로에서 운용 가능한지 판단해준다.

2. 문제 정의
드론 자율 비행에는 다음과 같은 문제가 존재한다.

비행 상태 변화에 따른 안정성의 비선형적 변화
센서 노이즈 및 외란(바람 등)에 대한 대응 어려움
경험 기반 운용으로 인한 판단 일관성 부족

3. 활용 데이터
Gazebo Sim 기반 Gazebo 환경에서 Iris quadrotor 모델을 사용하고, ArduPilot SITL과 연동하여 시뮬레이션을 수행하였다.
기체는 1.5 kg 질량의 표준 쿼드콥터 구조로 설정하였으며, IMU 및 GPS 등 기본 센서를 포함하였다.
배터리는 Gazebo Battery Plugin 기반 Linear battery model을 사용하였으며, 3S LiPo (11.1 V, 3.5 Ah)로 설정하였다.
평균 소비전력은 200 W로 가정하였으며, 이에 따른 이론적 배터리 런타임은 다음 수식과 같다.
	(capacity × voltage) / power_load = 약 11.7 min

(case 분류)
풍속 0, 3, 5 m/s (등속)
비행고도 50, 100 m (일부 200 포함)
이동거리 0, 500, 1000 m
회전각도 30, 90, 180 deg
waypoint 0~3 (일부 4 포함)

4. 역할 분담: 
데이터 수집 : 한재현

데이터 전처리: 
train data 21개 case 별 10개 sample
- 타겟 변수에 대해 노이즈(±2%) 반영: case 당 10개
	시뮬레이션 환경과 현실 환경간 불일치를 일부 완화하기 위한 목적으로 약한 noise 사용
- StandardScaler 적용.

test data 9개 case 별 1개 sample
- train data case와 독립적인 case로 구성

회귀 모델 4종 담당 (모델 평가 기준: MAE, R2_score)
전처리 방식 동일
RF : 이다현			XGBoost : 김동하
LR : 박상우			SVR : 이희동
hyper parameter tuning은 자율적

web : 김동하
등속, ideal한 상황가정, 실제보다 배터리 소모는 적음을 명시
이용자가 고도값 입력, 지도에 좌표(waypoint) 입력 하면
이를 이용해 총 이동거리와 회전각도를 구하고, 기상청 풍속 데이터를 이용하여 배터리 소모량 예측
